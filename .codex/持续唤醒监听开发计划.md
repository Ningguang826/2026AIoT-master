# 持续唤醒监听开发计划

## 1. 目标

在当前单轮实时语音闭环的基础上，升级为常驻待机系统：

`持续监听 -> 检测触发 -> 正式录音 -> ASR -> LLM -> TTS -> 播放 -> 回到待机`

这里的关键不是一开始就追求“最强唤醒词效果”，而是先做出一个稳定、可演示、可回到待机状态的持续监听版本。

## 2. 推荐分阶段实现

### 阶段 1：伪持续唤醒

目标：

先做出能循环待机的版本，不马上接复杂唤醒引擎。

建议方案：

1. 后台循环监听短窗音频
2. 每隔固定时间做一次轻量识别
3. 若文本中包含固定触发词，则进入正式对话流程

可选触发词示例：

- 小智
- 你好助手
- 开始对话

优点：

- 复用现有 DashScope ASR
- 开发简单
- 适合第一版演示

缺点：

- 不是严格意义的离线唤醒词
- 成本与延迟较高

### 阶段 2：真持续唤醒

目标：

在阶段 1 骨架稳定后，再考虑真正的 wake word 检测。

方向：

1. 板端接入专门唤醒词检测模块
2. 唤醒成功后才进入正式 ASR
3. TTS 播放完成后自动恢复监听

该阶段暂不作为当前第一优先级。

## 3. 状态机设计

持续监听必须使用状态机管理，避免录音、识别、播放互相冲突。

建议最小状态：

### 3.1 IdleListening

待机监听状态。

职责：

- 周期性采集短窗音频
- 判断是否出现触发词

进入条件：

- 系统启动后默认进入
- 对话结束后返回

退出条件：

- 识别到触发词

### 3.2 ActivatedRecording

正式录音状态。

职责：

- 录用户完整问题
- 停止待机监听
- 送入 ASR

退出条件：

- ASR 得到文本
- 超时未得到文本

### 3.3 Thinking

云端生成状态。

职责：

- 调用 DeepSeek
- 生成流式文本

退出条件：

- 开始进入 TTS 分句播放

### 3.4 Speaking

播报状态。

职责：

- TTS 分句合成
- 扬声器播放

退出条件：

- 全部播报完成

### 3.5 ErrorRecovery

异常恢复状态。

职责：

- 记录失败原因
- 清理资源
- 回到待机

## 4. 第一版最小实现原则

### 原则 1：先单线程主循环，再谈复杂线程池

当前项目不直接搬旧项目完整线程架构。

第一版持续监听建议：

- 一个主循环
- 一个状态变量
- 每个状态只做一件事

这样更容易调试。

### 原则 2：先用“短窗 ASR 触发”模拟唤醒

不要一开始就接复杂 wake word 引擎。

第一版只需要证明：

- 系统可以持续待机
- 触发后能进入对话
- 对话结束后能回待机

### 原则 3：录音与播放必须互斥

在 Speaking 状态下：

- 不允许继续录音
- 不允许继续待机识别

否则极易出现：

- TTS 被自己识别
- 设备资源冲突
- 状态混乱

## 5. 第一版任务拆解

### 任务 A：抽出状态机入口

新增建议文件：

- `wake_loop.py`

职责：

- 管理状态切换
- 调用现有 `RealtimeASR`
- 调用现有 `DeepSeekClient`
- 调用现有 `StreamingTTSPlayer`

### 任务 B：实现短窗监听

建议：

- 每次监听 1 到 2 秒
- 若识别文本含触发词，则进入正式录音

### 任务 C：正式录音流程

建议：

- 触发后进入 `--realtime-seconds 5` 类似的正式录音模式
- 获取用户问题文本

### 任务 D：回答并播报

建议：

- 复用当前流式 DeepSeek + 分句 TTS

### 任务 E：回待机

播报完成后：

- 重置状态
- 回到 IdleListening

## 6. 第一版验收标准

只要达到以下四条，就算第一版持续唤醒成功：

1. 程序启动后可持续运行，不自动退出
2. 说出触发词后能进入正式识别
3. 系统回答完成后能自动回待机
4. 连续 3 轮触发无严重资源冲突

## 7. 当前不做的事

第一版持续监听阶段暂不做：

- 多轮上下文记忆
- 复杂角色管理
- 视觉联动
- MQTT 上报
- 真正离线 wake word 引擎深度优化
- 复杂线程池中断管理

这些内容放到持续监听稳定以后。

## 8. 当前下一步

下一步开发顺序建议：

1. 先修复实时模式退出时的 Windows 句柄告警
2. 跑通 `--realtime-seconds 5`
3. 新增 `wake_loop.py`
4. 先做“短窗 ASR 触发词监听”
5. 再做自动回待机
