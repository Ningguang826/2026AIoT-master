## 今日完成

- 修正 `系统响应时延 t` 的计时口径，使日志更接近手机手测：从用户真正停声开始，到首个非静音 PCM 写入播放后端结束。
- 将 TTS 首包前导静音裁掉，并把播放缓冲压到 30ms，减少“日志已写入但耳朵还没听到”的偏差。
- 保留稳定的 `cosyvoice-v1` websocket TTS 路径，移除不可用的旁支 TTS 尝试，避免测试指标被污染。
- 接入并收紧本地端点投机式 LLM 预取：可以隐藏一部分 ASR 服务端句尾等待，但最终文本新增关键约束时不会复用错误预取。
- 固化固定提示音缓存：唤醒确认和未听清兜底短语在启动阶段就可预热，避免第一轮额外在线等待。
- 固化即时应答词缓存：识别到有效问题后，先播本地缓存应答词，再自然接入正文 TTS，降低 LLM 首 token 空窗带来的体感等待。
- 清洗 `.codex` 文档，使“固定提示音缓存”和“即时应答词缓存”的实现逻辑、计时口径和当前限制保持一致。
